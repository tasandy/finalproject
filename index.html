Install

In order to use this software, you will need to download following Python packages:

Numpy, Pandas, Scikit-Learn, and Flask. For example,

$ sudo apt-install flask

To run the software:

1. Clone or download the repo

2. Go to the folder

$ cd .../finalproject

3. Run the website.py file from the terminal and click "Let's Predict," which is at the top of the website

$ python website.py

Background and Context

Remember when movie tickets were only $10? Movie ticket prices have been on the rise and are currently priced at approximately $14 - not including popcorn and soda, which adds another $10. With a potential grand total of $24, it has become more necessary for us to make sure that we are watching a movie worth watching and spending so much money on.

We have found an existing data set containing over 5000 data points for movies in Kaggle. The data set contains 28 variables ranging from information such as the director name, main actors and the number of likes that each actor has on Facebook, the movie budget, genre, etc. The objective of our project is to create a software that will take user inputs for a movie and return the result whether user should definitely watch or not.

The main learning goals for this project is to use machine learning to come up with a methodology to predict future IMDb scores, which ties to utilization of Python to build predictive models for data analytics. Moreover, our group wanted to familiarize ourselves with different library packages that are essential to data analytics and understand the difficulty in creating accurate models for predictions, and what factors affect the accuracy. Overall, since this is software development class, we wanted to develop the software that can integrate the aspects of machine learning and software development.

Implementation

To filter, clean, and utilize the movies dataset to predict whether or not we recommend a user to watch a movie or not, we used Python packages including matplotlib, pandas, numpy and scikit-learn. We used matplotlib to visualize the dataset as a matrix plot, pandas to read the csv file and create dataframes, numpy to create arrays in the variables, and scikit-learn to test different statistical models. We used three predictive model algorithms including linear regression, logistic regression, and k-Nearest neighbors. We found that logistic regression returned the highest accuracy of approximately 67% with the following predictors: year, number of Facebook likes for the director, the first, second and third actors, the number of faces in the movie poster, content rating, budget and genre. Because our final predictive model utilizes a categorical response, we converted the movie ratings column:

  • Bad - Not recommended (0 - 5)
  • Good - Consider watching the movie (5 - 7)
  • Excellent - Go watch it! (7 - 10)

    • After building our final predictive model, we developed this software using Flask. We organized our code into three files that contains functions to clean and filter the data, predict the movie recommendation, and to suggest other movie recommendations based on user inputs. Our last file (website.py) calls the function from each of these and runs them together to deliver a prediction.

    Project Evolution

    Initially when we started this project, we were most concerned about how to clean and filter the data so that we can even begin to build a predictive model. Our dataset had 16 columns of 28 with missing values, with some columns as many as 500 missing values. We weighed the options: to delete all the records with missing values which would eliminate a lot of data, or to fill in the missing values with an average of the column which meant skewing the data - especially if the range is large. We proceeded with a combination of both.

    In addition, the predictor ‘genre’ was stored in concatenated form - multiple genres associated with a movie were stored in one cell. A few different ways we thought about proceeding were randomly picking one of the genres listed, selecting only the first one, or creating dummy variables for all the genres. After an exploration of Excel functions, we were able to convert a genre that looks like “[Action | Drama | Thriller]” into dummy variables.

    Lastly, we built our predictive model on the movies that were produced in year 2000 or after as well as in the English language to make the results more relevant to our interests. While we were coding the dataset, we came across a lot of unfamiliarities with the libraries used. For example, the concept of using a data frame and numpy array was new to us; thus when trying to simply prepare the data or building the model, we ran into numerous errors with the input type.

    Upon building a successful predictive model, we shifted our focus to building our software to recommend whether or not to view a movie given user inputs. Using flask and html template, we created a web page where users can input information on the movie they are interested in finding out the scores. After we retrieved the user input, we called in two functions to predict how good the movie is and recommend top 3 movies.
